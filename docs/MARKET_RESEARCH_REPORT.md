# 시장 조사 및 유사 프로그램 분석 리포트

> **Project:** WOORIDO (Community-First Funding Solution)
> **작성일:** 2025-12-23 (개정판)
> **분석 방법론:** 천재적 통찰 도출 공식(GI) × 다차원적 분석 프레임워크(MDA)
> **목적:** 시장 검증 및 **고객 수입·지출 데이터 분석 기반** 솔루션 전략 도출

---

## Executive Summary

본 리포트는 **1,220만 명**이 사용하는 모임통장 시장과 **7조 9천억 원** 규모의 팬덤/취향 시장을 분석하여, WOORIDO의 핵심 미션인 **"고객 수입·지출 데이터 분석 기반 지역 계모임 운영 솔루션"**의 시장 적합성을 검증한다.

**핵심 발견:**
1. 모임통장 시장은 **연 30% 성장**하지만, **신뢰 인프라는 0%** 성장
2. 기존 서비스들은 **"고객 재정 상황"**을 고려하지 않은 단순 통장 서비스
3. WOORIDO는 **고객 재정 분석 → 신뢰 점수 → 맞춤 계모임 추천**의 차별화된 파이프라인 제공

**핵심 차별화:**
```
기존 서비스:  통장 개설 → 돈 모으기 → 문제 발생 시 대응 (Reactive)
WOORIDO:     고객 재정 분석 → 신뢰 검증 → 맞춤 매칭 → 투명 운영 (Proactive)
```

## [2] 분석 방법론: 천재적 사고 공식 적용

### 2.1 천재적 통찰 도출 공식 (Genius Insight Formula)

```
GI = (O × C × P × S) / (A + B)

적용:
- O(관찰의 깊이): 9점 - 1,220만 사용자, 12조 원 규모 데이터 확보
- C(연결의 독창성): 9점 - 금융 + 커뮤니티 + 검색엔진 + 고객 재정 분석 통합
- P(패턴 인식): 9점 - "계주 리스크" + "재정 불일치" 패턴 발견
- S(종합적 사고): 9점 - 4대 코어 + 재정 분석 파이프라인

- A(고정관념): 2점 - "모임통장은 단순 통장" 공식 파괴
- B(편향): 2점 - 기술 중심이 아닌 사용자 재정 상황 중심

GI = (9 × 9 × 9 × 9) / (2 + 2) = 6,561 / 4 = **1,640.25점**
```

**해석:** 1,500점 이상은 **시장 정의(Market Defining)** 인사이트 수준

### 4.2 두 번째 결함: 재정 불일치 (Financial Mismatch)

**기존 서비스가 놓친 것:**

```
[기존 서비스의 가입 흐름]

사용자 A (월 500만원 수입) ──┐
사용자 B (월 200만원 수입) ──┼──▶ 같은 계모임 (10만원/월)
사용자 C (월 150만원 수입) ──┘         │
                                       ▼
                            사용자 C: 10만원이 부담 😰
                                       │
                                       ▼
                            3개월 후 납입 실패 → 중도 이탈
                                       │
                                       ▼
                            다른 멤버들 피해 + 신뢰 붕괴
```

**문제점:**
1. **재정 상황 파악 없음**: 가입 시 재정 능력 검증 부재
2. **무리한 가입**: 취향만 맞으면 재정 불일치해도 가입
3. **중도 이탈 리스크**: 재정적 부담으로 인한 이탈 빈발
4. **피해 전가**: 이탈자의 몫이 나머지 멤버에게 전가

```
│  ❌ 고객 재정 분석: 없음                                     │

└─────────────────────────────────────────────────────────────┘
```

| 항목 | 분석 |

### 8.3 Django 분석 로직 예시

```python
# woorido_analytics/services/financial_analyzer.py
import pandas as pd
import numpy as np

class FinancialAnalyzer:
    """사용자 재정 프로필 분석"""

    SAFE_CONTRIBUTION_RATIO = 0.15  # 저축 가능액의 15%
    TARGET_SAVINGS_RATE = 0.25       # 목표 저축률 25%

    def analyze_profile(self, data: dict) -> dict:
        income = data['income']
        expense = data['expense']
        savings_capacity = data['savings_capacity']

        # 저축률 계산
        savings_rate = savings_capacity / income if income > 0 else 0

        # 재정 건전성 점수 (0-100)
        health_score = min(100, (savings_rate / self.TARGET_SAVINGS_RATE) * 100)

        # 적정 월 납입금 계산
        appropriate_contribution = savings_capacity * self.SAFE_CONTRIBUTION_RATIO

        # 리스크 등급 분류
        if health_score >= 90:
            risk_level = 'low'
        elif health_score >= 70:
            risk_level = 'medium'
        else:
            risk_level = 'high'

        return {
            'appropriateContribution': int(appropriate_contribution),
            'financialHealthScore': round(health_score, 1),
            'riskLevel': risk_level,
            'savingsRate': round(savings_rate * 100, 1),
            'analysis': self._generate_analysis(health_score, risk_level)
        }

    def _generate_analysis(self, score: float, risk: str) -> str:
        if risk == 'low':
            return f"재정 건전성이 우수합니다 ({score}점). 다양한 계모임 참여가 가능합니다."
        elif risk == 'medium':
            return f"재정 건전성이 보통입니다 ({score}점). 적정 납입금 범위 내에서 참여를 권장합니다."
        else:
            return f"재정 건전성 개선이 필요합니다 ({score}점). 소액 계모임부터 시작해보세요."
```
