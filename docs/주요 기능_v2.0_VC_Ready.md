> **커뮤니티 기반 계모임 플랫폼입니다.**
>
> 토스가 "편리함"을 제공했다면, WOORIDO는 "신뢰"를 제공합니다.

**수수료 정당화 논리:**
```
토스 모임통장 (무료):
  └─ 계주가 독단 출금 가능 = 먹튀 위험 100%

WOORIDO (유료):
  └─ 투표 기반 출금 승인 = 먹튀 위험 0%
  └─ 보증금 락 시스템 = 이탈 방지
```

## 🛡️ Trust Triangle: 기술적 신뢰 구현

### 핵심 차별화 요소

```
                    ┌──────────┐
                    │  장부    │
                    │  투명화  │
                    │ (Django) │
                    └────┬─────┘
                        /│\
                       / │ \
                      /  │  \
                     /   │   \
            ┌───────┐  ┌─┴─┐  ┌───────────┐
            │ 선충전 │──│신뢰│──│ 결제 감시 │
            │  락   │  │   │  │  다각화   │
            │(Lock) │  └───┘  │(Consensus)│
            └───────┘          └───────────┘

           1개월 보증금        모든 지출
           선입금으로          멤버 투표
           이탈 방지          승인 필요
```

### 기술적 구현 (Demo Day 시연 가능)

#### 1. 선충전 락 (Deposit Lock)

```
사용자 시나리오:
1. 유저가 모임 가입 클릭
2. 시스템: "100,000원 보증금이 1개월간 락됩니다"
3. 가용 잔액 → 락 잔액 이동 (즉시)
4. 매월 1일 자동 납입

기술 스펙:
- Spring Boot Transaction Management
- 락 해제 조건: 완주/탈퇴/강제퇴출
- 실시간 잔액 업데이트 (헤더 표시)
```

#### 2. 결제 감시 다각화 (Consensus Pay)

```
투표 알고리즘:
- 일반 지출 (<10만원): 과반수 (50%)
- 대형 지출 (≥10만원): 2/3 (67%)
- 기권 표 = 분모에서 제외

예시:
모임원 10명, 지출 150,000원
├─ 찬성 7명, 반대 2명, 기권 1명
├─ 승인 기준: 9명 × 67% = 6명
└─ 결과: 승인 ✅ (7 ≥ 6)

Demo Day 시연:
→ 실시간 투표 UI + 바코드 생성 애니메이션
```

#### 3. 장부 투명화 (Django Analytics)

```
Django pandas 분석:
- 월별 지출 통계
- 카테고리별 비율
- 지출 트렌드 예측
- 다음 달 예상 지출

Demo Day 시연:
→ Recharts 라이브 차트 + "지출 30% 감소" 인사이트
```

### Q8. MyBatis vs JPA 왜 MyBatis?

**A**: 복잡한 장부 쿼리 최적화
```sql
-- 예시: 카테고리별 월별 지출 집계 (5 테이블 Join)
SELECT c.name, SUM(l.amount), AVG(l.amount)
FROM ledger l
JOIN category c ON l.category_id = c.id
JOIN group_member gm ON l.group_id = gm.group_id
WHERE l.created_at BETWEEN ? AND ?
GROUP BY c.name
HAVING SUM(l.amount) > 100000;
```
→ JPA로 표현 시 N+1 문제 + 성능 저하
→ MyBatis 네이티브 SQL이 더 효율적
