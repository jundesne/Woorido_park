| 문서 | 보증금 | 가입 시 필요 금액 |
|------|--------|------------------|
| **최종 설계** ✅ | **1개월치** | 2개월치 (1개월 납입 + 1개월 보증금) |
| VC 피치덱 ❌ | 3개월치 | 4개월치 (1개월 납입 + 3개월 보증금) |

**조치**: VC 피치덱 수정 필요

### 2. AI 신뢰점수 우선순위 불일치 ⚠️

| 문서 | AI 신뢰점수 | Django 분석 | 우선순위 |
|------|------------|------------|----------|
| **최종 설계** ✅ | 언급 없음 | 2순위 (추천 알고리즘) | SNS → 장부 → 투표 |
| VC 피치덱 ❌ | Phase 0 최우선 | Phase 0 최우선 | Django 먼저 |

**문제점**:
1. 최종 설계에는 "AI 신뢰점수"가 **아예 없음**
2. 최종 설계에서 Django는 **2순위** (있으면 좋음)
3. VC 피치덱은 Django를 **Phase 0** (최우선)으로 설정

**결론**: 최종 설계 기준으로 가되, VC 피치덱의 AI 신뢰점수는 **Future Enhancement**로 분류

## 🚀 Future Enhancements (VC 피치덱 내용)

### Phase 1: Post-Demo (3개월 후)

**우선순위 높음**:

#### 1. 재정 프로필 입력 🌟
```
목적: 사용자 온보딩 시 재정 정보 수집
구현:
- Frontend: 온보딩 폼 (월 수입, 지출, 저축 가능액)
- Backend: user_financial_profiles 테이블
- 난이도: 중 (2주)
```

#### 2. Django 재정 분석 🌟
```
목적: pandas로 적정 월 납입금 계산
구현:
- Django 프로젝트 생성
- pandas 재정건전성 계산 로직
- Spring ↔ Django HTTP 연동
- 난이도: 중 (3주)
```

#### 3. 기본 신뢰점수 시스템
```
목적: 완주율, 투표 참여율 기반 점수 (재정 분석 제외)
구현:
- user_trust_scores 테이블
- 계산식: completionRate × 0.5 + voteParticipation × 0.5
- AI/머신러닝 없이 단순 계산
- 난이도: 하 (1주)
```

**총 기간**: 6주
