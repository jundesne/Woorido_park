```python

재정건전성 = min(100, (저축률 / 0.25) × 100)

# 머신러닝 모델 (추후)

```

**차별점**:
- 카카오/토스: 신용점수 연동 없음
- WOORIDO: **자체 신뢰점수 = Data Moat** 🔒


### 4.3 Unit Economics

**고객 1명 기준** (Y2 기준):

```
LTV (Lifetime Value)
= 월 수수료 × 평균 체류 기간 × 계모임 참여 수
= (10만원 × 3%) × 24개월 × 2개
= 14.4만원

CAC (Customer Acquisition Cost)
= 마케팅 비용 / 신규 가입자
= 연 20억 / 5만명
= 4만원

LTV/CAC Ratio = 14.4 / 4 = 3.6 ✅ (목표 >3)

Payback Period = 4만원 / (14.4만원 / 24개월) = 6.7개월 ✅
```

**건강한 지표**:
- LTV/CAC > 3 ✅
- Payback < 12개월 ✅
- Monthly Churn < 5% (Target: 3%)

#### **VC에게 전달할 메시지**

**Q: "왜 AI를 MVP에 안 넣나?"**
**A**:
1. **Cold Start Problem**: AI는 데이터가 있어야 의미있음. 첫 1,000명은 AI 없이도 가입 가능
2. **MVP는 핵심 가치 증명**: "보증금 Lock + 투표 + 투명 장부"만으로도 먹튀 방지 가능
3. **데이터 수집이 먼저**: 3개월간 재정 데이터 수집 → 그 후 AI 학습
4. **올바른 순서**: 사용자 확보 → 데이터 축적 → AI 고도화 (역순은 실패)

**Q: "카카오가 베끼면?"**
**A**:
1. **특허 준비 중**: Phase 2에서 신뢰점수 알고리즘 특허 출원
2. **First Mover Advantage**: 2년 선점 시간 (MVP 출시 → AI 고도화)
3. **Data Moat**: 재정 데이터는 복사 불가능 (사용자가 직접 입력)
4. **Exit 전략**: 카카오가 베끼기보다 **인수**가 더 빠름 (M&A 타겟)
